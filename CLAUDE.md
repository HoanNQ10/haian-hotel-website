# Claude Instructions for Haian Hotel Website

This file contains instructions for AI assistance on this project.

## Project Overview
- **Project Name**: Hải An Villa Hotel Website
- **Type**: Personal project - Small family-run hotel
- **Purpose**: Static one-page hotel landing page
- **Technology**: Astro + Tailwind CSS (decided)
- **No Backend**: No database, CMS, admin dashboard, booking system, or payment integration

## Key Rules
- Keep everything simple and suitable for a static hotel landing page
- Hotel name: Hải An Villa Hotel (family-run, small hotel)
- Use docs/content.md as the source of truth for hotel information
- Use docs/requirements.md as the source of truth for functional requirements
- Use docs/design.md as the source of truth for visual direction
- Unknown information should be marked as TBD
- Do not create backend, database, CMS, admin dashboard, booking system, or payment features
- Room types are classified by single/double and balcony availability only
- Additional features (BBQ yard, attic, views, special layout) are optional tags/notes, not main room types

## Website Structure
- **Sections**: Header, Hero, Introduction, Rooms, Amenities, Gallery, Location, Contact, Footer
- **Main Actions**: Call phone number, Contact via Zalo, Open Google Map
- **Out of Scope**: Backend, database, CMS, admin, login, online booking, payment, room availability management

## Development Guidelines
- Follow the documentation in docs/ folder
- Validate changes against requirements and content docs
- Keep code simple and maintainable
- Edit Astro components in `src/` folder
- Use Tailwind CSS utility classes for styling
- No additional packages unless necessary

## Workflow

### 1. Brainstorming
- Clarify the goal
- Identify options
- Explain trade-offs
- Do not modify files

### 2. Planning
- Break the selected option into small tasks
- List files to create or modify
- Explain verification steps
- Do not modify files until approval

### 3. Execution
- Implement only the approved task
- Keep changes small
- Do not refactor unrelated files
- Do not add unnecessary libraries

### 4. Verification
- Report changed files
- Explain how to run and test

- Recommend next step

## Project Structure
```
src/
├── components/      # Reusable Astro components (Button, Card, Header, etc.)
├── layouts/         # Base Layout.astro with meta tags
├── pages/           # index.astro (landing page with 9 sections)
└── styles/          # globals.css (Tailwind imports + global styles)

docs/                # Documentation (design.md, content.md, requirements.md)
tailwind.config.mjs  # Design tokens (colors, spacing)
astro.config.mjs     # Astro config
package.json         # Dependencies & scripts
dist/                # Build output (generated by `npm run build`)
```

## Key Design Decisions
- **Framework**: Astro for lightweight static generation, excellent for SEO
- **Styling**: Tailwind CSS with custom design tokens from docs/design.md
- **Components**: Minimal, reusable Astro components (no JavaScript frameworks)
- **Fonts**: System fonts initially (upgrade to Playfair Display + Inter later)
- **Images**: Placeholder images currently (need real hotel photos)
